The application shall be developed natively for Android OS.
The application shall target a minimum Android API level of 26 (Android 8.0 Oreo).
The application shall be primarily developed using Kotlin.
The application shall utilize a secure RESTful API to communicate with the banking backend system.
The application shall implement user authentication via username/password and optional biometric recognition (fingerprint/face unlock).
The application shall display real-time account balances and transaction history.
The application shall enable funds transfers between user's own accounts.
The application shall enable funds transfers to pre-registered third-party beneficiaries.
The application shall support bill payment functionality.
The application shall allow users to view and manage (e.g., block/unblock) their linked debit/credit cards.
The application shall adhere to industry-standard security practices, including OWASP Mobile Security Testing Guide (MSTG) guidelines.
All data transmitted between the app and backend services shall be secured using TLS 1.2 or higher.
The user interface shall conform to Google Material Design 3 guidelines.
The application shall support push notifications for critical account activities and announcements.
The application shall include a branch and ATM locator feature utilizing device location services.
The application shall be optimized for performance, ensuring quick loading times and smooth navigation.
The application shall be published to the Google Play Store as an Android App Bundle (AAB).
The application shall provide in-app user registration and onboarding workflows.
The application shall support multi-factor authentication (MFA) for sensitive transactions.
The application shall support dark theme capabilities.
